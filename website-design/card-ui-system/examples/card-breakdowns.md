# Card Layout Breakdowns

This document illustrates how to apply the **Three-Point Rule** and **Visual
Hierarchy** to common card types.

## 1. Product Listing Card (E-commerce)

High-density cards focused on quick comparison and conversion.

### The Breakdown

- **Point 1: The Anchor (Visual)**
  - Product Image (1:1 Aspect Ratio)
  - Badge: "New Arrival" or "Sale" (Top-left overlay)
- **Point 2: The Context (Primary Content)**
  - Brand Name (Uppercase, small, muted)
  - Product Title (Bold, H3)
  - Price (High contrast, slightly larger than title)
  - Rating (Stars + count)
- **Point 3: The Target (Primary Action)**
  - "Add to Cart" Button (Full width, Primary color)

### Why it works

The user identifies the product (Anchor), checks the price and reputation
(Context), and has an immediate path to purchase (Target).

---

## 2. Article/Blog Preview Card

Content-focused cards designed for scannability and discovery.

### The Breakdown

- **Point 1: The Anchor (Visual)**
  - Featured Image (16:9 Aspect Ratio)
- **Point 2: The Context (Primary Content)**
  - Category Tag (Accent color, underlined)
  - Article Title (H3, serif or bold sans-serif)
  - Excerpt (Max 3 lines, clamped)
  - Metadata: Date & Reading Time (Muted text)
- **Point 3: The Target (Primary Action)**
  - "Read Article" Text Link with Arrow icon (→)

### Why it works

The image catches the eye (Anchor), the title and category establish relevance
(Context), and the link provides the "Next Step" (Target).

---

## 3. Feature Highlight Card (SaaS/Marketing)

Icon-driven cards used to explain benefits or features.

### The Breakdown

- **Point 1: The Anchor (Visual)**
  - Decorative Icon (Accent circle background)
- **Point 2: The Context (Primary Content)**
  - Feature Name (H3, centered or left-aligned)
  - Description (Short, concise paragraph)
- **Point 3: The Target (Primary Action)**
  - "Learn More" Stretched Link (The entire card is clickable)

### Why it works

The icon sets the theme (Anchor), the text explains the benefit (Context), and
the full-card clickability reduces interaction cost (Target).

## Visual Comparison: Desktop vs. Mobile

| Element        | Desktop Grid               | Mobile Stack           |
| :------------- | :------------------------- | :--------------------- |
| **Layout**     | 3-4 Columns                | 1-2 Columns            |
| **Image**      | Side-by-side or Top        | Usually Top            |
| **CTA**        | Visible on Hover or Always | Always Visible (Touch) |
| **Title Size** | 1.25rem - 1.5rem           | 1.1rem - 1.25rem       |
