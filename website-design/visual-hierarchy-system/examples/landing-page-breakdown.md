# Example: Landing Page Hierarchy Breakdown

This example demonstrates how to apply the Visual Hierarchy System to a standard
SaaS landing page hero and feature section.

## 1. Hero Section Breakdown

### Elements Inventory & Priority

1. **H1 Headline:** (Priority 1) - The Core Value Prop.
2. **Subheadline:** (Priority 2) - Context and supporting info.
3. **Primary CTA:** (Priority 1) - The main action (Sign Up).
4. **Product Screenshot:** (Priority 2) - Visual proof of functionality.
5. **Nav Links:** (Priority 3) - Site navigation.
6. **Secondary CTA:** (Priority 2) - Low-friction alternative (Watch Video).

### Visual Weight Assignment

- **Priority 1 (H1):** Largest font size (64px), bold weight, high-contrast
  color against background.
- **Priority 1 (CTA):** Vibrant brand color background, white text, 16px of
  padding, placed directly under the headline.
- **Priority 2 (Subhead):** Medium font size (20px), neutral gray color,
  positioned between H1 and CTA.
- **Priority 3 (Nav):** Smallest text (14px), muted color, placed at the top
  periphery.

---

## 2. Feature Grid Breakdown

When presenting a 3-column feature grid, the hierarchy ensures the user scans
the benefits before the details.

### Hierarchy Levers Applied:

- **Icon (Level 2):** Uses a brand color to draw the eye to the start of each
  block.
- **Feature Title (Level 2):** Uses an H3 tag, bold weight, slightly larger than
  body text.
- **Feature Description (Level 3):** Standard body size, slightly muted color to
  recede behind the title.
- **"Learn More" Link (Level 3):** Text link with an arrow icon, no background
  color, to avoid competing with the Hero CTA.

---

## 3. The "Squint Test" Result

When viewing the layout blurred:

1. The **Primary CTA button** and **H1 Headline** should stand out as the
   strongest blobs.
2. The **Product Screenshot** should be the next recognizable shape.
3. All other text and nav links should blend into the background.
