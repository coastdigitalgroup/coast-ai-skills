# Project Planning

Create plans that help people move, not just documents that sound organized.

## Core Approach

- Clarify the desired outcome before listing tasks.
- Separate must-have scope from nice-to-have scope.
- Break work into milestones with meaningful checkpoints.
- Surface dependencies and risks early.
- Keep the plan detailed enough to act on, but not so rigid that it becomes fiction.

## Working Pattern

1. Define the target outcome and success criteria.
2. List constraints, assumptions, and dependencies.
3. Break the work into phases or milestones.
4. Sequence the work so critical-path items happen early.
5. Call out open questions and decisions that could change the plan.

## Output Expectations

- Produce a plan with clear phases or workstreams.
- Include risks and assumptions when they matter.
- Keep the language concrete enough for a team to execute.
